# Demo 设备列表管理 — 产品设计文档

## 概述

对 Blue SDK Android Demo 应用进行交互改造，从"扫描即连接第一个设备"升级为"设备列表管理"模式。用户可绑定多台 LX-PD02 设备，每次选择一台连接使用。

**定位：** 面向集成方开发者的 SDK 演示 Demo
**约束：** SDK 单设备连接架构不变，Demo 层维护绑定列表

---

## 页面结构

### 页面 1：设备列表页（新 MainActivity）

**布局：**
- 顶部标题栏："Blue SDK Demo" + 右侧"+"添加按钮
- 设备卡片列表（垂直排列）
- 底部版本号

**设备卡片内容：**
- 设备名称（如 "LX-PD02-A1B2"）
- 设备 MAC 地址（灰色小字）
- 在线状态标签：🟢 在线 / ⚪ 离线
- 连接状态：已连接（蓝色高亮边框）/ 未连接

**卡片交互：**
- 点击在线设备 → 连接并进入设备控制页
- 点击离线设备 → Toast 提示"设备不在附近"
- 长按 → 弹出菜单：删除设备

**页面行为：**
- 进入页面时自动发起 BLE 扫描（5秒）
- 将扫描到的设备与本地绑定列表比对，更新在线状态
- 扫描期间顶部显示"扫描中..."指示器
- 已连接的设备卡片显示蓝色高亮边框 + "已连接"标签

---

### 页面 2：添加设备页（ScanActivity）

**触发：** 点击列表页右上角"+"按钮

**布局：**
- 顶部标题："扫描设备" + 返回按钮
- 扫描动画/进度指示
- 扫描到的设备列表（实时更新）
- 底部"停止扫描"按钮

**设备条目内容：**
- 设备名称
- MAC 地址
- RSSI 信号强度图标
- "绑定"按钮

**交互流程：**
1. 进入页面自动开始扫描（超时 15 秒）
2. 发现新设备实时追加到列表（已绑定设备不显示）
3. 用户点击"绑定" → 设备加入本地绑定列表 → 自动连接 → 跳转设备控制页
4. 扫描超时或手动停止 → 停留在当前页面，用户可重新扫描

---

### 页面 3：设备控制页（DeviceControlActivity，原 MainActivity 功能）

**触发：** 从设备列表点击已在线设备

**行为：**
- 将当前 MainActivity 的所有控制功能（音频设置、闹钟管理、用药记录等）迁移到此页面
- 顶部显示当前连接设备名称 + 返回按钮
- 返回时不断开连接（保持连接状态）
- 设备断开时自动返回设备列表页

---

## 数据模型

### BoundDevice（本地持久化）

| 字段 | 类型 | 说明 |
|------|------|------|
| deviceId | String | 设备 MAC 地址（唯一标识） |
| deviceName | String | 设备广播名称 |
| bindTime | Long | 绑定时间戳 |
| lastConnectedTime | Long | 最后连接时间戳 |

### 运行时状态（不持久化）

| 字段 | 类型 | 说明 |
|------|------|------|
| isOnline | Boolean | 当前扫描到 = 在线 |
| isConnected | Boolean | 当前已连接 |
| rssi | Int | 当前信号强度 |

---

## 存储方案

使用 SharedPreferences + JSON 序列化：
- Key: `"bound_devices"`
- Value: `List<BoundDevice>` 的 JSON 字符串

理由：设备数量少（≤10），结构简单，无需 SQLite。

## 核心交互流程

### 流程 1：首次使用（无绑定设备）

```
打开 App → 设备列表页（空态）
         → 提示"点击 + 添加您的第一台设备"
         → 用户点击 + → 进入扫描页
         → 发现设备 → 点击绑定 → 自动连接认证 → 进入控制页
```

### 流程 2：日常使用（有绑定设备）

```
打开 App → 设备列表页
         → 自动扫描 5 秒更新在线状态
         → 用户点击在线设备 → 连接认证 → 进入控制页
```

### 流程 3：添加新设备

```
设备列表页 → 点击 + → 扫描页
           → 发现新设备（排除已绑定）
           → 点击绑定 → 保存到本地 + 连接认证 → 进入控制页
```

### 流程 4：删除设备

```
设备列表页 → 长按设备卡片 → 确认弹窗"确定删除此设备？"
           → 确认 → 如果该设备当前已连接则先断开
           → 从本地列表移除
```

## 连接管理策略

| 场景 | 行为 |
|------|------|
| 点击已在线设备 | 显示 loading → 连接 + 自动认证 → 成功跳转控制页 |
| 连接/认证失败 | Toast 错误信息，留在列表页 |
| 控制页返回列表 | 保持连接，卡片显示"已连接" |
| 点击另一台设备 | 先断开当前连接 → 连接新设备 |
| 设备意外断开 | 从控制页自动返回列表页 + Toast 提示 |
| App 退到后台 | 保持连接（不主动断开） |
| App 被杀死 | 下次打开重新连接 |

## 验收标准

### AC-1：设备列表展示
- 打开 App 显示设备列表页
- 已绑定设备按最后连接时间倒序排列
- 无设备时显示空态引导

### AC-2：在线状态检测
- 进入列表页自动扫描 5 秒
- 扫描到的已绑定设备标记为"在线"
- 扫描期间显示加载指示器

### AC-3：添加设备
- 点击 + 进入扫描页
- 扫描结果排除已绑定设备
- 点击绑定后设备持久化到本地列表
- 绑定后自动连接并进入控制页

### AC-4：连接设备
- 点击在线设备显示 loading 并发起连接
- 连接+认证成功跳转控制页
- 连接失败显示错误提示

### AC-5：删除设备
- 长按设备弹出删除确认
- 确认后从列表移除
- 如设备已连接则先断开

### AC-6：设备控制页
- 包含原 MainActivity 全部控制功能
- 顶部显示当前设备名称
- 设备断开时自动返回列表页

---

## 技术约束

- SDK `BlueSDK` 单例，单设备连接，不改动 SDK 层代码
- 设备列表管理完全在 Demo app 层实现
- 沿用当前 Demo 的纯代码 UI 风格（无 XML 布局）
- 沿用当前深色主题配色
- 密钥认证逻辑不变（phoneMac / fixedAuthKey / customPhoneMac）
