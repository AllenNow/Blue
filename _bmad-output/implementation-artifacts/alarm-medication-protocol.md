# 闹钟与服药记录完整协议文档

## 帧数据包结构

| 序号 | 字节数 | 字段 | 说明 |
|:---:|:---:|------|------|
| 0 | 1 | 帧头 | 0x55 |
| 1 | 1 | 帧头 | 0xAA |
| 2 | 1 | 协议版本号 | 0x00 |
| 3 | 1 | 命令字 CMD | 0x06=APP下发, 0x07=设备上报 |
| 4 | 1 | 数据长度 Len | 高8位 |
| 5 | 1 | 数据长度 Len | 低8位 |
| 6~6+Len-1 | Len | 数据 | data[] |
| 6+Len | 1 | CRC8 | 从帧头第一字节累加和的结果对256求余 |

## 一、闹钟设置/上报（DPID 0x66~0x6C）

### 1.1 APP 下发设置闹钟

示例：设置闹钟1为 12:00，每天重复

```
55 AA 00 06 00 0B 66 00 00 07 01 0C 00 7F 00 00 00 09
```

按 data 偏移逐字节：

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x66 | DPID（闹钟序号：0x66=闹钟1, 0x67=闹钟2 ... 0x6C=闹钟7） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x07 | 内部数据长度=7 |
| [4] | 0x01 | 闹钟使能位（0x00=关闭, 0x01=开启） |
| [5] | 0x0C | 小时=12 |
| [6] | 0x00 | 分钟=0 |
| [7] | 0x7F | 周期使能位（bit0=周日, bit1=周一 ... bit6=周六，0x7F=每天） |
| [8] | 0x00 | 保留 |
| [9] | 0x00 | 保留 |
| [10] | 0x00 | 保留 |

### 1.2 设备端设置闹钟上报

示例：设备上设置闹钟1为 07:00，每天重复

```
55 AA 00 07 00 0B 66 00 00 07 01 07 00 7F 00 00 00 05
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x66 | DPID（闹钟1） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x07 | 内部数据长度=7 |
| [4] | 0x01 | 闹钟使能位（0x01=开启） |
| [5] | 0x07 | 小时=7 |
| [6] | 0x00 | 分钟=0 |
| [7] | 0x7F | 周期使能位（0x7F=每天） |
| [8] | 0x00 | 触发标志（0x00=未触发, 0x01=已触发） |
| [9] | 0x00 | 响铃状态（0x00=响铃中, 0x01=响铃结束） |
| [10] | 0x00 | 用药记录标志（0x01=未取药标识, 0x02=已有取药记录） |

### 1.3 设备端删除闹钟上报

示例：删除闹钟1

```
55 AA 00 07 00 0A 66 00 00 06 00 18 3C 00 00 00 0D
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x66 | DPID（闹钟1） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x06 | 内部数据长度=6 |
| [4] | 0x00 | 闹钟使能位（0x00=关闭） |
| [5] | 0x18 | 小时=24（超出范围表示已删除） |
| [6] | 0x3C | 分钟=60（超出范围表示已删除） |
| [7] | 0x00 | 周期使能位=0 |
| [8] | 0x00 | 触发标志 |
| [9] | 0x00 | 响铃状态 |

### 1.4 APP 下发删除闹钟

示例：删除闹钟1

```
55 AA 00 06 00 0B 66 00 00 07 FF FF FF FF FF FF FF XX
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x66 | DPID（闹钟1） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x07 | 内部数据长度=7 |
| [4] | 0xFF | 全FF表示删除 |
| [5] | 0xFF | 全FF |
| [6] | 0xFF | 全FF |
| [7] | 0xFF | 全FF |
| [8] | 0xFF | 全FF |
| [9] | 0xFF | 全FF |
| [10] | 0xFF | 全FF |

## 二、闹钟响铃事件流程

### 2.1 设备闹钟时间到开始响铃上报

**第一帧：上报取药结果通知（DPID=0x6F），用于 APP 推送通知**

```
55 AA 00 07 00 05 6F 04 00 01 01 80
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x6F | DPID（用药结果通知） |
| [1] | 0x04 | type（枚举类型） |
| [2] | 0x00 | len high |
| [3] | 0x01 | len low = 1 |
| [4] | 0x01 | 通知类型（0x01=通知闹钟开始响铃） |

**第二帧：上报开始响铃的闹钟信息并更改闹钟状态（DPID=0x68，以闹钟3为例）**

```
55 AA 00 07 00 0B 68 00 00 07 01 00 12 7F 01 00 01 14
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x68 | DPID（闹钟3） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x07 | 内部数据长度=7 |
| [4] | 0x01 | 闹钟使能位（0x01=开启） |
| [5] | 0x00 | 小时=0 |
| [6] | 0x12 | 分钟=18 |
| [7] | 0x7F | 周期使能位（0x7F=每天） |
| [8] | 0x01 | 触发标志（0x01=闹钟已触发） |
| [9] | 0x00 | 响铃状态（0x00=正在响铃） |
| [10] | 0x01 | 用药记录标志（0x01=未取药标识） |

### 2.2 设备闹钟时间超时上报

**第一帧：上报取药结果通知（DPID=0x6F）**

```
55 AA 00 07 00 05 6F 04 00 01 02 81
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x6F | DPID（用药结果通知） |
| [1] | 0x04 | type（枚举类型） |
| [2] | 0x00 | len high |
| [3] | 0x01 | len low = 1 |
| [4] | 0x02 | 通知类型（0x02=通知错过服药时间） |

**第二帧：上报闹钟响铃结束的闹钟信息并更改闹钟状态（DPID=0x68）**

```
55 AA 00 07 00 0B 68 00 00 07 01 00 12 7F 01 01 02 16
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x68 | DPID（闹钟3） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x07 | 内部数据长度=7 |
| [4] | 0x01 | 闹钟使能位（0x01=开启） |
| [5] | 0x00 | 小时=0 |
| [6] | 0x12 | 分钟=18 |
| [7] | 0x7F | 周期使能位（0x7F=每天） |
| [8] | 0x01 | 触发标志（0x01=闹钟已触发） |
| [9] | 0x01 | 响铃状态（0x01=响铃结束/停止） |
| [10] | 0x02 | 用药记录标志（0x02=已有取药记录） |

### 2.3 用户及时按取药键取药上报

**第一帧：上报取药结果通知（DPID=0x6F）**

```
55 AA 00 07 00 05 6F 04 00 01 03 81
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x6F | DPID（用药结果通知） |
| [1] | 0x04 | type（枚举类型） |
| [2] | 0x00 | len high |
| [3] | 0x01 | len low = 1 |
| [4] | 0x03 | 通知类型（0x03=通知服药成功） |

**第二帧：上报闹钟响铃结束的闹钟信息并更改闹钟状态（DPID=0x68）**

```
55 AA 00 07 00 0B 68 00 00 07 01 00 12 7F 01 01 02 16
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x68 | DPID（闹钟3） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x07 | 内部数据长度=7 |
| [4] | 0x01 | 闹钟使能位（0x01=开启） |
| [5] | 0x00 | 小时=0 |
| [6] | 0x12 | 分钟=18 |
| [7] | 0x7F | 周期使能位（0x7F=每天） |
| [8] | 0x01 | 触发标志（0x01=闹钟已触发） |
| [9] | 0x01 | 响铃状态（0x01=响铃结束/停止） |
| [10] | 0x02 | 用药记录标志（0x02=已有取药记录） |

> **注意：** 超时上报（2.2）和取药上报（2.3）的闹钟帧完全相同，区分依靠 0x6F 通知帧中的类型值（0x02=超时, 0x03=取药）。

## 三、用药记录上报（DPID 0x65）

### 3.1 设备上报用药记录

示例：闹钟3，2025年11月1日，设定时间00:18，取药时间00:18，状态=按时取药

```
55 AA 00 07 00 0F 65 00 00 0B 68 07 E9 0B 01 00 12 00 12 01 00 0A
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x65 | DPID（用药记录） |
| [1] | 0x00 | type high |
| [2] | 0x00 | type low |
| [3] | 0x0B | 内部数据长度=11 |
| [4] | 0x68 | 闹钟DP点（0x66=闹钟1, 0x67=闹钟2, 0x68=闹钟3 ... 0x6C=闹钟7） |
| [5] | 0x07 | 年高位 |
| [6] | 0xE9 | 年低位 → 0x07E9=2025年 |
| [7] | 0x0B | 月=11 |
| [8] | 0x01 | 日=1 |
| [9] | 0x00 | 闹钟设定小时=0 |
| [10] | 0x12 | 闹钟设定分钟=18 |
| [11] | 0x00 | 实际取药小时=0 |
| [12] | 0x12 | 实际取药分钟=18 |
| [13] | 0x01 | 用药状态（0x01=按时取药, 0x02=超时取药, 0x03=漏服, 0x04=提前取药） |
| [14] | 0x00 | 提前标志（0x00=未提前取药, 0x01=提前取药） |

## 四、用药结果通知（DPID 0x6F）

### 4.1 通知类型定义

| data[4] | 含义 |
|:---:|------|
| 0x00 | 等待闹钟响铃 |
| 0x01 | 通知闹钟开始响铃 |
| 0x02 | 通知错过服药时间 |
| 0x03 | 通知服药成功 |

### 4.2 帧格式

```
55 AA 00 07 00 05 6F 04 00 01 XX CRC
```

| data[] | 值 | 说明 |
|:---:|:---:|------|
| [0] | 0x6F | DPID（用药结果通知） |
| [1] | 0x04 | type（枚举类型） |
| [2] | 0x00 | len high |
| [3] | 0x01 | len low = 1 |
| [4] | 0xXX | 通知类型（见上表） |

## 六、事件回调对应关系

| 帧来源 | DPID | 职责 | SDK 回调 |
|--------|:---:|------|---------|
| 闹钟 DPID | 0x66~0x6C | 闹钟配置/状态变更，UI 更新 | `onAlarmUpdated` |
| 用药结果通知 | 0x6F | APP 推送通知（响铃/超时/取药） | `onMedicationNotification` |
| 用药记录 | 0x65 | 精确用药记录入库 + 用药结果 | `onMedicationResult` + `onMedicationRecordReported` |

## 七、周期使能位定义（data[7] / byte2）

| 位 | 值 | 星期 |
|:---:|:---:|:---:|
| bit0 | 0x01 | 周日 |
| bit1 | 0x02 | 周一 |
| bit2 | 0x04 | 周二 |
| bit3 | 0x08 | 周三 |
| bit4 | 0x10 | 周四 |
| bit5 | 0x20 | 周五 |
| bit6 | 0x40 | 周六 |

默认值 0x7F = 每天（bit0~bit6 全为1）

常用组合：
- 0x7F = 每天
- 0x3E = 工作日（周一~周五，bit1~bit5）
- 0x41 = 周末（周六+周日，bit0+bit6）
- 0x0E = 周一+周二+周三（bit1+bit2+bit3）

## 八、完整事件流程时序

```
设备闹钟时间到：
  ① 0x6F data[4]=0x01（通知响铃开始）
  ② 闹钟DPID data[8]=0x01, data[9]=0x00（触发+响铃中）

用户按键取药：
  ③ 0x6F data[4]=0x03（通知服药成功）
  ④ 闹钟DPID data[8]=0x01, data[9]=0x01, data[10]=0x02（触发+结束+有记录）
  ⑤ 0x65 完整用药记录（状态=0x01 按时取药）

超时未取药：
  ③ 0x6F data[4]=0x02（通知错过服药）
  ④ 闹钟DPID data[8]=0x01, data[9]=0x01, data[10]=0x02（触发+结束+有记录）
  ⑤ 0x65 完整用药记录（状态=0x03 漏服）

超时后取药：
  ③ 0x6F data[4]=0x02（通知错过服药）
  ④ 闹钟DPID（触发+结束）
  ... 用户稍后按键 ...
  ⑤ 0x6F data[4]=0x03（通知服药成功）
  ⑥ 闹钟DPID data[10]=0x02（更新有记录标志）
  ⑦ 0x65 完整用药记录（状态=0x02 超时取药）
```
