# BlueSDK 协议参考文档

**设备型号**: LX-PD02  
**协议版本**: 0x00  
**文档版本**: 1.0  
**最后更新**: 2026-05-07

---

## 1. 帧格式

### 1.1 通用帧结构

| 序号 | 字节数 | 字段 | 说明 |
|------|--------|------|------|
| 0 | 1 | 帧头 | 固定 `0x55` |
| 1 | 1 | 帧头 | 固定 `0xAA` |
| 2 | 1 | 协议版本号 | 固定 `0x00` |
| 3 | 1 | 命令字 CMD | 具体帧类型 |
| 4 | 1 | 数据长度 Len 高字节 | |
| 5 | 1 | 数据长度 Len 低字节 | |
| 6~6+Len-1 | Len | 数据 | |
| 6+Len | 1 | CRC8 | 从帧头第一字节累加和对 256 求余 |

### 1.2 CRC8 算法

```
crc8 = (byte[0] + byte[1] + ... + byte[6+Len-1]) % 256
```

**验证示例：**

| 帧 | 数据（不含CRC） | CRC8 |
|----|----------------|------|
| 查询设备信息 | `55 AA 00 01 00 00` | `0x00` |
| 密钥包 | `55 AA 00 00 00 02 07 74` | `0x7C` |
| 时间同步请求 | `55 AA 00 E1 00 01 00` | `0xE1` |
| 设置闹钟1(12:00) | `55 AA 00 06 00 0B 66 00 00 07 01 0C 00 7F 00 00 00` | `0x09` |

## 2. CMD 命令字

| CMD | 值 | 方向 | 说明 |
|-----|----|------|------|
| AUTH_KEY | `0x00` | APP→设备 | 密钥认证 |

## 3. DPID 功能字节

数据段第一字节为 DPID，标识数据类型。

| DPID | 值 | 方向 | 说明 |
|------|----|------|------|
| ALARM_RECORD | `0x65` | 双向 | 闹钟记录（含时间戳和用药状态）|
| ALARM_1 | `0x66` | 双向 | 闹钟1 |
| ALARM_2 | `0x67` | 双向 | 闹钟2 |
| ALARM_3 | `0x68` | 双向 | 闹钟3（同时用于用药事件上报）|
| ALARM_4 | `0x69` | 双向 | 闹钟4 |
| ALARM_5 | `0x6A` | 双向 | 闹钟5 |
| ALARM_6 | `0x6B` | 双向 | 闹钟6 |
| ALARM_7 | `0x6C` | 双向 | 闹钟7 |
| TYPE_OF_SOUND | `0x6D` | 双向 | 声音类型（1-静音 2-A 3-B）|
| VOLUME_LEVEL | `0x6E` | 双向 | 音量（1-低 2-中 3-高）⚠️ 待确认 |
| SOUND_TYPE_SETTING | `0x6F` | 双向 | 铃声类型设置 ⚠️ 待确认 |
| ALERT_DURATION_SETTING | `0x70` | 双向 | 提醒持续时间 ⚠️ 待确认 |
| RESTORE_FACTORY | `0x71` | APP→设备 | 恢复出厂设置 |
| TIME_FORMAT | `0x73` | 双向 | 时制（0-12小时 1-24小时）|
| SILENCE | `0x74` | 双向 | 静音（0-关 1-开）|
| LOW_BAT | `0x75` | 设备→APP | 低电量 |

> ⚠️ **注意**：DPID `0x6E`/`0x6F`/`0x70` 的实际用途与协议文档注释存在歧义，以示例帧为准，待硬件方最终确认。

## 4. 密钥认证

### 4.1 密钥计算算法

```
key[i] = (phoneMac[i] + deviceMac[i]) & 0xFF   (i = 0..5)
```

**示例：**
- 手机 MAC：`C7 50 B2 AA C3 F3`
- 设备 MAC：`A6 C0 82 00 A1 C2`
- 密钥：`6D 10 34 AA 64 B5`

### 4.2 认证帧

**APP 发送：**
```
55 AA 00 00 00 06 [key0] [key1] [key2] [key3] [key4] [key5] [crc8]
```

**设备应答（成功）：**
```
55 AA 00 00 00 01 01 01
```

**设备应答（失败）：**
```
55 AA 00 00 00 01 00 00
```

## 5. 闹钟数据格式

### 5.1 闹钟配置（下发/上报）

数据段格式（11字节）：

| 字节 | 说明 |
|------|------|
| 0 | DPID（0x66~0x6C）|
| 1-2 | 保留（`0x00 0x00`）|
| 3 | 固定 `0x07` |
| 4 | 固定 `0x01` |
| 5 | 小时（0~23）|
| 6 | 分钟（0~59）|
| 7 | 星期掩码（bit0=周日...bit6=周六，`0x7F`=每天）|
| 8-9 | 提前状态高低字节 |
| 10 | 提前取药状态 |

**删除闹钟**：字节 5~10 全填 `0xFF`

### 5.2 用药事件上报（DPID=0x68）

| byte10 值 | 含义 |
|-----------|------|
| `0x01` | 按时取药（TAKEN）|
| `0x02` | 超时取药（TIMEOUT）|
| `0x03` | 漏服（MISSED）|
| `0x04` | 提前取药（EARLY）|
