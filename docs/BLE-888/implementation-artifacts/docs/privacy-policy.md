# BlueSDK 隐私政策说明

**版本**: 1.0.0  
**生效日期**: 2026-05-07  
**适用范围**: BlueSDK iOS / Android

---

## 一、概述

BlueSDK（以下简称"本 SDK"）是专为 LX-PD02 智能药盒设计的蓝牙通信 SDK。本文档说明 SDK 本身的数据处理行为，以及集成方（第三方开发者）在使用本 SDK 时应承担的隐私合规责任。

> **重要声明**：本 SDK **不收集、不存储、不上传**任何用户个人数据。所有数据处理责任由集成方的 APP 承担。

---

## 二、SDK 数据处理行为

### 2.1 SDK 不处理的数据

| 数据类型 | 说明 |
|---------|------|
| 用户身份信息 | SDK 不收集姓名、手机号、身份证等 |
| 用药记录 | SDK 仅将设备上报的用药事件通过回调传递给 APP，不做任何存储 |
| 位置信息 | SDK 不主动获取位置，Android 扫描 BLE 需要位置权限但不使用位置数据 |
| 设备 MAC 地址 | 仅在内存中用于密钥计算，不持久化，不上传 |
| 健康数据 | SDK 不处理任何健康相关数据 |

### 2.2 SDK 处理的数据（仅内存态）

| 数据 | 用途 | 存储方式 | 保留时长 |
|------|------|---------|---------|
| 手机 MAC 地址 | 计算设备绑定密钥 | 仅内存 | SDK 销毁时清除 |
| 设备 MAC 地址 | 计算设备绑定密钥 | 仅内存 | SDK 销毁时清除 |
| BLE 连接状态 | 维护连接状态机 | 仅内存 | SDK 销毁时清除 |
| 指令队列 | 管理下行指令发送 | 仅内存 | 断开连接时清除 |

### 2.3 网络请求

**本 SDK 不包含任何网络请求能力**，所有通信仅限本地蓝牙（BLE）通道。

## 三、权限使用说明

### 3.1 iOS 权限

| 权限 | 用途 | 是否必须 |
|------|------|---------|
| `NSBluetoothAlwaysUsageDescription` | 扫描和连接 LX-PD02 蓝牙设备 | ✅ 必须 |

**说明**：iOS 13.0+ 仅需蓝牙权限，不需要位置权限。

### 3.2 Android 权限

| 权限 | 适用版本 | 用途 | 是否必须 |
|------|---------|------|---------|
| `BLUETOOTH` | Android 6~11 | 蓝牙基础功能 | ✅ 必须 |
| `BLUETOOTH_ADMIN` | Android 6~11 | 蓝牙管理 | ✅ 必须 |
| `ACCESS_FINE_LOCATION` | Android 6~11 | BLE 扫描（系统要求，不使用位置数据）| ✅ 必须 |
| `BLUETOOTH_SCAN` | Android 12+ | BLE 设备扫描 | ✅ 必须 |
| `BLUETOOTH_CONNECT` | Android 12+ | BLE 设备连接 | ✅ 必须 |

> **重要说明（Android 6~11）**：`ACCESS_FINE_LOCATION` 是 Android 系统对 BLE 扫描的强制要求，SDK **不会**读取或使用用户的地理位置信息。建议在 APP 隐私政策中向用户说明此权限的实际用途。

## 四、集成方责任

集成本 SDK 的第三方开发者（集成方）需承担以下隐私合规责任：

### 4.1 数据存储

- 用药记录、闹钟配置等数据由 APP 负责存储
- 应遵守《个人信息保护法》（PIPL）、GDPR 等适用法规
- 健康数据属于敏感个人信息，需获得用户明确同意

### 4.2 隐私政策

集成方的 APP 隐私政策应包含：

```
本应用使用蓝牙功能连接 LX-PD02 智能药盒设备，用于：
- 设置用药提醒闹钟
- 接收用药事件通知
- 记录用药历史

蓝牙连接过程中，本应用会访问设备的蓝牙功能，
不会收集您的地理位置信息。
```

### 4.3 用户授权

- 在请求蓝牙权限前，向用户说明权限用途
- Android 6~11 申请位置权限时，明确告知"仅用于蓝牙扫描，不获取位置"
- 提供权限拒绝后的降级处理方案

### 4.4 数据安全

- 用药记录等健康数据传输应使用 HTTPS 加密
- 本地存储应使用加密存储方案
- 不得将用药数据用于广告投放或数据交易

---

## 五、儿童隐私

本 SDK 不专门面向儿童设计。如集成方的 APP 面向 14 岁以下儿童，需额外遵守相关儿童隐私保护法规，并获得监护人同意。
