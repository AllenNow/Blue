# 中文语言偏好设置

## 核心规则

所有与用户的交互、文档编写、代码注释和输出内容都应该使用中文作为主要语言。

## 适用范围

1. **用户交互**
   - 所有回复和对话使用中文
   - 技术解释和说明使用中文
   - 错误信息和警告使用中文

2. **文档编写**
   - README 文件使用中文
   - 技术文档使用中文
   - 设计文档使用中文
   - 需求文档使用中文
   - 用户故事使用中文

3. **代码注释**
   - 函数和类的注释使用中文
   - 复杂逻辑的说明使用中文
   - TODO 和 FIXME 注释使用中文

4. **输出内容**
   - 日志信息使用中文
   - 用户界面文本使用中文
   - 提示信息使用中文

## 例外情况

以下情况可以使用英文：

1. **代码元素**
   - 变量名、函数名、类名等标识符（遵循编程规范）
   - 第三方库和框架的 API 调用
   - 技术关键词和专有名词（如 API、HTTP、JSON 等）

2. **技术术语**
   - 当中文翻译可能造成歧义时，可以使用英文原文并附带中文解释
   - 例如："使用 RESTful API（表述性状态转移应用程序接口）"

3. **代码示例**
   - 代码本身使用标准编程语言语法
   - 但代码的说明和注释使用中文

## 实施指南

- 在生成任何文档或回复之前，确认使用中文
- 保持专业和清晰的中文表达
- 使用准确的技术术语中文翻译
- 必要时可以在括号中附加英文原文以避免歧义
