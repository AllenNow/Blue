# BlueSDK 隐私政策

> 最后更新：2025-06-27

---

## 1. 概述

BlueSDK 是一个**纯本地蓝牙通信 SDK**，完整封装 LX-PD02 智能药盒的 BLE 通信协议。本 SDK：

- ❌ **不进行任何网络通信**
- ❌ **不上传任何数据到服务器**
- ❌ **不包含任何第三方 SDK 或分析服务**
- ❌ **不收集用户个人信息**

---

## 2. 数据处理说明

### 2.1 SDK 处理的数据

| 数据类型 | 用途 | 存储方式 | 生命周期 | 加密 |
|----------|------|----------|----------|------|
| phoneMac（6 字节随机标识） | BLE 配对认证密钥计算 | Android: SharedPreferences (应用私有)<br>iOS: Keychain | 卸载 APP 或调用 clearBinding() 时清除 | Android: 应用沙盒隔离<br>iOS: Keychain 加密 |
| 设备 MAC 地址 | 密钥计算 + 设备识别 | 仅运行时内存 | 断开连接后释放 | — |
| 闹钟配置数据 | 通过回调传递给 APP | SDK 不存储 | 回调完成后释放 | — |
| 用药记录数据 | 通过回调传递给 APP | SDK 不存储 | 回调完成后释放 | — |
| SDK 运行日志 | 调试排查 | 运行时内存（环形缓冲区，最多 1000 条） | APP 进程结束后丢失 | 密钥值自动脱敏 |

### 2.2 SDK 不处理的数据

- ❌ 用户个人信息（姓名、手机号、身份证、邮箱等）
- ❌ 地理位置信息（仅使用位置权限满足系统 BLE 扫描要求，不读取 GPS 坐标）
- ❌ 设备标识符（IMEI、IDFA、Android ID、广告 ID 等）
- ❌ 网络相关数据（IP 地址、网络类型等）
- ❌ 用户行为数据（使用习惯、点击事件等）
- ❌ 生物特征数据
- ❌ 健康数据（用药记录通过回调传递给 APP，SDK 本身不解读也不存储）

## 3. 权限使用说明

### 3.1 Android

| 权限 | 用途 | 必要性 | 备注 |
|------|------|--------|------|
| `BLUETOOTH` | BLE 通信 | 必须 | 基础蓝牙功能 |
| `BLUETOOTH_ADMIN` | BLE 扫描 | Android 11 及以下必须 | |
| `ACCESS_FINE_LOCATION` | BLE 扫描 | Android 6~11 必须 | 系统要求，SDK 不读取位置 |
| `BLUETOOTH_SCAN` | BLE 扫描 | Android 12+ 必须 | 配置 `neverForLocation` |
| `BLUETOOTH_CONNECT` | BLE 连接 | Android 12+ 必须 | |

> `ACCESS_FINE_LOCATION` 声明了 `usesPermissionFlags="neverForLocation"`，SDK 内部不调用任何位置 API。

### 3.2 iOS

| 权限 | 用途 | 必要性 | 备注 |
|------|------|--------|------|
| `NSBluetoothAlwaysUsageDescription` | BLE 通信 | 必须 | 蓝牙连接与数据交换 |
| `bluetooth-central` (Background Mode) | 后台接收设备上报 | 可选 | 用于后台接收用药提醒 |

## 4. 数据安全措施

- **密钥脱敏**：所有日志输出中，认证密钥值均以 `***` 替代，绝不以明文出现
- **沙盒隔离**：phoneMac 存储在应用私有区域，其他应用无法访问
- **iOS Keychain**：iOS 端使用 Keychain 存储 phoneMac，由系统加密保护
- **无远程访问**：SDK 不包含任何后门、远程控制或数据回传机制
- **无动态代码加载**：不包含热更新、远程配置等动态行为

---

## 5. 数据共享

本 SDK **不与任何第三方共享数据**：
- 不向广告平台发送数据
- 不向分析服务发送数据
- 不向云服务器发送数据
- 不与其他 SDK 共享数据

---

## 6. 合规声明

| 法规/平台 | 合规状态 | 说明 |
|-----------|---------|------|
| 中国《个人信息保护法》(PIPL) | ✅ | 遵循最小必要原则，不采集个人信息 |
| 欧盟 GDPR | ✅ | 不处理个人数据，无需用户同意 |
| Google Play 数据安全政策 | ✅ | 不触发 "Data collected" 声明 |
| Apple App Store 隐私标签 | ✅ | 可声明 "Data Not Collected" |
| HIPAA（美国健康信息） | ✅ | SDK 不存储健康数据 |

---

## 7. 接入方责任

SDK 通过回调将用药记录等数据传递给接入方 APP。**接入方有责任**：
- 对用药记录的存储和处理遵循相关隐私法规
- 在 APP 隐私政策中如实披露数据处理行为
- 为用户提供数据删除能力
